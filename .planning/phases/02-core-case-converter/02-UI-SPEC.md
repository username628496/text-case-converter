## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tab internal padding, badge padding |
| sm | 8px | Compact element spacing, button padding vertical |
| md | 16px | Default element spacing, card padding, input padding |
| lg | 24px | Section padding, gap between major tool sections |
| xl | 32px | Layout gaps (main content top padding) |
| 2xl | 48px | Major section breaks (FAQ, related tools separation) |
| 3xl | 64px | Page-level spacing (not used in Phase 2 tool UI) |

Exceptions:
- Textarea minimum height: 160px (not on scale — functional requirement for usable text input area)
- Tab bar touch target minimum height: 44px (accessibility — WCAG 2.5.5 minimum touch target size)
- AdSense slot heights: 90px (header/below-tool), 250px (sidebar) — fixed by INFRA-02, not adjustable

Source: Spacing scale is the standard 8-point system. Exceptions derived from REQUIREMENTS.md CORE-01 (textarea usability) and CORE-08 (accessibility compliance).

|------|------|--------|-------------|------|

Source: Existing `src/app/[locale]/page.tsx` uses `text-3xl font-bold` for h1 and `text-lg` for subheading — confirmed. Mono output requirement from CORE-01 (textarea for converted text output). Two weights only: 400 + 600.

## Component Inventory

### ToolPage (Client Component — `'use client'`)

The primary interactive component. Contains all state: inputText, activeMode, isCopied.

**Structure:**
```
<section>
  <h1>  — Tool title (from i18n)
  <p>   — Tool description (from i18n)
  <div class="tab-bar">
    7 × <button role="tab"> — one per case mode
  </div>
  <textarea>  — unified input/output textarea (single textarea pattern)
  <div class="toolbar">
    <span>  — character count display
    <span>  — word count display
    <button> — Copy CTA
    <button> — Clear
  </div>
</section>
```

**Tab bar — 7 modes:**
1. Sentence case
2. lower case
3. UPPER CASE
4. Capitalized Case
5. aLtErNaTiNg CaSe
6. Title Case
7. iNVERSE cAsE

Active tab: `border-b-2 border-zinc-900 dark:border-zinc-100 font-semibold text-zinc-900 dark:text-zinc-100`
Inactive tab: `border-b-2 border-transparent text-zinc-500 dark:text-zinc-400 hover:text-zinc-700 dark:hover:text-zinc-300`

**Textarea:**
- Min-height: 160px
- Resize: vertical only (`resize-y`)
- Classes: `w-full min-h-[160px] resize-y rounded-md border border-zinc-200 dark:border-zinc-700 bg-white dark:bg-zinc-950 px-4 py-3 text-sm font-mono text-zinc-900 dark:text-zinc-100 placeholder:text-zinc-400 focus:outline-none focus:ring-2 focus:ring-zinc-900 dark:focus:ring-zinc-100`
- Placeholder text: from i18n key (e.g. "Type or paste your text here…")

**Toolbar row:**
- Character count: `text-sm text-zinc-500`
- Word count: `text-sm text-zinc-500`
- Copy button (normal): `px-4 py-2 rounded-md bg-zinc-900 dark:bg-zinc-100 text-white dark:text-zinc-900 text-sm font-semibold hover:bg-zinc-700 dark:hover:bg-zinc-300 transition-colors`
- Copy button (copied state): `px-4 py-2 rounded-md bg-zinc-500 text-white text-sm font-semibold` — label changes to "Copied!" for ~1500ms then reverts
- Clear button: `text-sm text-red-600 dark:text-red-500 hover:underline` — text-only, no background fill

### FAQ Section (Server Component)

Rendered below the ToolPage. Required for JSON-LD HowTo schema validity (CORE-07).

**Structure:**
```
<section aria-labelledby="faq-heading">
  <h2 id="faq-heading">  — "Frequently Asked Questions" (from i18n)
  <dl>
    <dt> — Question
    <dd> — Answer
  </dl>
</section>
```

Classes: Section has `mt-12 pt-8 border-t border-zinc-200 dark:border-zinc-800`; h2 uses heading style; dt uses `text-base font-semibold text-zinc-900 dark:text-zinc-100 mt-6`; dd uses `mt-1 text-base text-zinc-600 dark:text-zinc-400`

### Related Tools (Server Component)

Rendered below FAQ. Required to reduce bounce rate (CORE-06).

**Structure:**
```
<section aria-labelledby="related-heading">
  <h2 id="related-heading">  — "Related Tools" (from i18n)
  <ul role="list">
    N × <li><a href="..."> — one per related tool
  </ul>
</section>
```

Classes: Section has `mt-12`; links use `inline-flex items-center gap-2 text-base text-zinc-700 dark:text-zinc-300 hover:text-zinc-900 dark:hover:text-zinc-100 underline underline-offset-2`

### JSON-LD Script (Server Component)

Rendered as first child of the page section, before ToolPage. Inline `<script type="application/ld+json">` — no `next/script`.

These constraints come from Phase 1 implementation (INFRA-02) and must not be violated:

| Slot | Placement | Min-height | Classes |
|------|-----------|------------|---------|

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Tab list semantics | `role="tablist"` on tab bar container, `role="tab"` on each button, `aria-selected="true/false"`, `aria-controls="tab-panel"` |
| Tab panel | `role="tabpanel"` on the textarea wrapper, `id` matching tab `aria-controls` |
| Keyboard navigation | Tab bar supports arrow key navigation between tabs (left/right); Enter/Space activates tab |
| Textarea label | `<label>` visually hidden (`sr-only`) with `for` linking to textarea `id`; label text: "Enter text to convert" |
| Copy button | `aria-label="Copy converted text"` when label is just "Copy" |
| Clear button | `aria-label="Clear all text"` |
| Count display | `aria-live="polite"` on the count container so screen readers announce updates |
| Dark mode | Implemented via OS preference (`prefers-color-scheme`) — no ARIA needed |
| Minimum touch target | 44px height on all tab buttons and action buttons |
| Focus ring | `focus:outline-none focus:ring-2 focus:ring-zinc-900 dark:focus:ring-zinc-100` on all interactive elements |
