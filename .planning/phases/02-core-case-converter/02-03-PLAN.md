Create `src/components/ToolPage.tsx` as a Client Component (`'use client'`).

**Imports:**
```typescript
'use client'
import { useState, useRef } from 'react'
import { useTranslations } from 'next-intl'
import { transformText, CASE_MODES, type CaseMode } from '@/lib/case-transforms'
```

**State:**
- `inputText: string` (default: `''`)
- `activeMode: CaseMode` (default: `'sentence'`)
- `isCopied: boolean` (default: `false`)

**Derived values (computed inline — no useMemo, React Compiler handles it):**
- `outputText = inputText ? transformText(inputText, activeMode) : ''`
- `charCount = outputText.length`
- `wordCount = outputText.trim() ? outputText.trim().split(/\s+/).length : 0`

**Behavior:**
- Textarea is a SINGLE unified field. User types/pastes into it. The textarea VALUE is `outputText` (the transformed text). When user types, `onChange` sets `inputText` to the raw input. However — this creates a UX issue where the cursor jumps. Better approach: store `inputText` as the raw input, display `outputText` in the textarea, but make the textarea display the transformed output. When mode changes, transform the current inputText. When user types, update inputText and immediately show transformed output.
- Actually, the simplest UX: textarea shows the CONVERTED output. User types into textarea → `onChange` captures the raw typed value as `inputText`, then `outputText` is computed from `inputText` + `activeMode`. The textarea `value` is set to `outputText`. This means when user types "hello" with mode=upper, they see "HELLO" appear character by character. When they switch mode to lower, the text becomes "hello".
- Wait — this will fight with cursor position. Better pattern: use TWO logical states but ONE textarea. Set textarea `value={outputText}`. On change, capture `e.target.value` as the new inputText (this is what the user actually typed, before transform). The transform runs on the raw input.

**Actually simplest correct approach:**
- `inputText` stores what the user typed (raw)
- Textarea `value` = `outputText` (transformed)
- `onChange`: set `inputText = e.target.value` (capture raw keystroke). Since the textarea shows the output, the user sees the transform happen live.
- But WAIT: if textarea value is outputText and user types a char, the onChange value will be the outputText + new char, not the raw input. This is wrong.

**Correct approach (matching competitor sites like convertcase.net):**
- The textarea is an INPUT field. It shows what the user types AS-IS (raw).
- The OUTPUT is also shown in the SAME textarea. When the user selects a mode, the textarea content is REPLACED with the transformed version.
- So: textarea value = outputText. onChange sets inputText to e.target.value. outputText = transformText(inputText, activeMode). When mode changes, inputText stays the same, outputText updates.
- The key insight: typing into a textarea whose value is `transformText(rawInput, mode)` means each keystroke the user sees the transformed version. If mode is "upper", typing "h" shows "H". This is correct behavior — instant conversion per CORE-02.

**Implementation:**
```tsx
const handleChange = (e: React.ChangeEvent<HTMLTextAreaElement>) => {
  setInputText(e.target.value)
}

const handleModeChange = (mode: CaseMode) => {
  setActiveMode(mode)
}

const handleCopy = async () => {
  if (!outputText) return
  try {
    await navigator.clipboard.writeText(outputText)
    setIsCopied(true)
    setTimeout(() => setIsCopied(false), 1500)
  } catch {
    // Clipboard API failure is silent per UI-SPEC
  }
}

const handleClear = () => {
  setInputText('')
}
```

import { getTranslations } from 'next-intl/server'

interface FAQProps {
  locale: string
}

export default async function FAQ({ locale }: FAQProps) {
  const t = await getTranslations({ locale, namespace: 'faq' })

  // Access items — next-intl v4 supports raw() for arrays

  return (

        {t('heading')}
      </h2>
      <dl>
        {items.map((item, i) => (
          <div key={i}>
            <dt className="text-base font-semibold text-zinc-900 dark:text-zinc-100 mt-6">
              {item.q}
            </dt>
            <dd className="mt-1 text-base text-zinc-600 dark:text-zinc-400">

**2. Create `src/components/RelatedTools.tsx`** (Server Component — no 'use client'):

```typescript
import { getTranslations } from 'next-intl/server'
import { tools, type Tool } from '@/lib/tools'

interface RelatedToolsProps {
  locale: string
  relatedSlugs: string[]
}

export default async function RelatedTools({ locale, relatedSlugs }: RelatedToolsProps) {
  const t = await getTranslations({ locale, namespace: 'relatedTools' })
  const tTools = await getTranslations({ locale, namespace: 'tools' })

  const relatedTools = relatedSlugs
    .map(slug => tools.find(tool => tool.slug === slug))
    .filter((tool): tool is Tool => tool !== undefined)

  if (relatedTools.length === 0) return null

  const prefix = locale === 'en' ? '' : '/vi'

  return (
    <section aria-labelledby="related-heading" className="mt-12">
      <h2 id="related-heading" className="text-xl font-semibold leading-tight text-zinc-900 dark:text-zinc-100">
        {t('heading')}
      </h2>
      <ul role="list" className="mt-4 space-y-2">
        {relatedTools.map(tool => {
          // Extract the nested key from i18nKey like "tools.caseConverter" -> "caseConverter"
          const key = tool.i18nKey.replace('tools.', '')
          return (
            <li key={tool.slug}>
              <a
                href={`${prefix}/${tool.slug}/`}
                className="inline-flex items-center gap-2 text-base text-zinc-700 dark:text-zinc-300 hover:text-zinc-900 dark:hover:text-zinc-100 underline underline-offset-2"
              >
                {tTools(`${key}.name`)}
              </a>
            </li>
          )
        })}
      </ul>
    </section>
  )
}
```
