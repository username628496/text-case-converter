2. Constant `CASE_MODES` array of objects:
```ts
export const CASE_MODES: { id: CaseMode; badge: string; labelKey: string }[] = [
  { id: 'sentence', badge: 'Sc', labelKey: 'tool.modes.sentence' },
  { id: 'lower', badge: 'lc', labelKey: 'tool.modes.lower' },
  { id: 'upper', badge: 'UC', labelKey: 'tool.modes.upper' },

```

3. Individual pure functions:
- `toSentenceCase(text: string): string` — Lowercase all, then capitalize first char after `.`, `!`, `?` followed by whitespace, plus capitalize first char of string. Must handle multiple sentences.
- `toLowerCase(text: string): string` — `text.toLowerCase()`
- `toUpperCase(text: string): string` — `text.toUpperCase()`
- `toCapitalizedCase(text: string): string` — Split on whitespace, capitalize first letter of each word, lowercase rest.
- `toAlternatingCase(text: string): string` — Iterate chars: index 0 lowercase, index 1 uppercase, index 2 lowercase, etc. Non-letter chars still consume an index position.
- `toTitleCase(text: string): string` — Capitalize every word except skip list `['a','an','the','of','in','and','or','but','for','nor']` unless it's the first word. Lowercase the rest of each word.
- `toInverseCase(text: string): string` — Flip each letter's case. Non-letters unchanged.

4. Dispatcher `transformText(text: string, mode: CaseMode): string` — switch on mode, call appropriate function. Return empty string for empty input.

Create `src/lib/case-transforms.test.ts` with tests covering all behaviors listed above. Use vitest or the project's test runner. If no test runner configured, use a simple node-runnable test file with assertions that throws on failure.
