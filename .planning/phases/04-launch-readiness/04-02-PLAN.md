**If CWV gates failed (LCP >= 2500ms, CLS >= 0.1, or INP >= 200ms):**
Apply fixes based on what Lighthouse flagged. Common fixes for Next.js 16 SSG sites:

- **LCP too high**: Check font loading strategy in layout.tsx — ensure `next/font` is used with `display: swap`. Check if large images are above the fold without `priority` prop. Check for render-blocking CSS in globals.css.
- **CLS too high**: Check for elements that shift during load — AdSense placeholder divs should already have `min-height` reserved (90px header, 250px sidebar, 90px below-tool). Check for images without explicit width/height. Check for fonts causing layout shift (FOUT).
- **INP too high**: Check for heavy client-side JS in tool components. Ensure React Compiler is handling memoization (do NOT add manual useMemo/useCallback per project convention). Check for blocking event handlers.

After applying fixes:
1. Run `npm run build` to verify the build still succeeds with all pages statically generated
2. Push to main to trigger a new Vercel deployment
3. Wait for deployment to complete (check via `npx vercel ls` or the Vercel dashboard)
4. Re-run the specific failed Lighthouse audit(s): `npx lighthouse <url> --output=json --chrome-flags="--headless=new" --only-categories=performance`
5. Verify the fixed metrics now pass thresholds
6. Update 04-01-audit-results.md with the new scores
